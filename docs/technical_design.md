# 技術仕様書

## 1. 開発環境・技術スタック

* **ゲームエンジン:** Flutter + Flame
* **対象プラットフォーム:** PC (Steam / Windows, Mac)
* **推奨コントローラー:** ゲームパッド または キーボード
* **アーキテクチャ方針:** AIエージェント（Cursor等）にコードを生成させやすいよう、GUIに依存せずDartコードベースでコンポーネントを構築する。

## 2. アセット・描画仕様

* **基本解像度:** `640 × 360`（少し高精細なドット絵・ピクセルアートベース）。Flameの `CameraComponent` を使用してこのアスペクト比と解像度を維持・スケーリングする。
* **画像形式:** 背景完全透過の `PNG` 形式。スプライトシート化し、`SpriteAnimationComponent` で制御。ディレクトリ: `assets/images/`
* **音声形式:** BGMは `MP3 / OGG`、SEは `WAV` を推奨。ディレクトリ: `assets/audio/`
* **マップデータ:** 巨大な1枚マップを構築するため、外部ツールである `Tiled` (TMX形式) を使用し、Flameの `flame_tiled` パッケージ経由で読み込むことを想定。

## 3. コアクラス設計案（AI生成用）

* `ChaosPlayer`: プレイヤーの描画、アニメーション制御、物理演算（重力・移動・慣性）を担当。
* `InputManager`: キーボード/ゲームパッドの物理的な入力を検知する。
* `ChaosController`: 10秒タイマーを管理し、`InputManager`から受け取った入力を現在のモードに応じて「Jump, Attack, Dash」などの論理アクションに変換（マッピング）する頭脳。
* `WeaponSystem`: 8方向射撃、チャージ状態の管理、必殺技ゲージの管理を担当するコンポーネント。
* `HologramUI`: `ChaosPlayer`に追従し、現在のマッピング状態を視覚的に描画するUIコンポーネント。
* `RestPoint`: プレイヤーが接触（インタラクト）した際に、`ChaosController`のタイマーとマッピングを初期化し、リスポーン地点を更新するコンポーネント。
